## [BACKLOG-001] Trending 태그 기능
* **카테고리:** 프론트엔드
* **위치:** 메인 페이지 > 검색창 하단
* **설명:** 인기 검색 챔피언 태그를 표시하여 빠른 접근을 제공하는 기능.
  * 디자인 HTML에는 `Trending: 리신, 니달리, 그레이브즈` 태그가 검색창 하단에 존재함.
  * 현재 API 스펙에 trending 데이터를 내려주는 엔드포인트/필드가 없음.
* **필요 작업:**
  1. API 스펙에 trending 데이터 추가 (별도 엔드포인트 or 기존 API 1 응답에 포함)
  2. 프론트엔드에 `TrendingTags` 컴포넌트 구현
  3. 태그 클릭 시 해당 챔피언 검색 or 상세 페이지로 이동
* **디자인 참고:** `design/jungle_clear_design/메인페이지/code.html` 61~66번 라인

## [BACKLOG-002] 영상 조회수(Views) 표기
* **카테고리:** 프론트엔드 / 백엔드
* **위치:** 상세 페이지 > Video Player 하단
* **설명:** 현재 재생 중인 영상의 조회수를 Video Player 하단에 표기하는 기능.
  * 디자인 HTML에는 `14.2k views`로 표기됨.
  * 현재 API 2의 `videos` 배열에 `views` 필드가 없음.
* **필요 작업:**
  1. API 2의 videos 배열 각 아이템에 `views` (Number) 필드 추가
  2. 프론트엔드에서 views 포매팅 (예: 14200 → "14.2k") 및 표기
  3. YouTube Data API 연동 or 자체 집계 방식 결정
* **디자인 참고:** `design/jungle_clear_design/상세페이지/code.html` 108~111번 라인

## [BACKLOG-005] 마스터 티어 수집 대상 확장 **(필수)**
* **카테고리:** 백엔드 / 데이터 파이프라인
* **설명:** 현재 수집 대상은 그랜드마스터/챌린저(약 1,000명)로 한정. Rate Limit 여유가 확인되면 마스터 티어까지 확장하여 데이터 풀을 넓힌다.
* **전제 조건:**
  1. Phase 2 구현 완료 후 Test Key 기준 API 호출량 측정 및 Rate Limit 여유 확인.
  2. 또는 Riot에서 Production Key 발급 후 실서비스 환경에서 테스트.
* **필요 작업:**
  1. 마스터 티어 포함 시 예상 API 호출량 산정
  2. Rate Limit 내에서 소화 가능한지 검증
  3. 수집 대상 티어 설정을 환경 변수 또는 설정값으로 분리
* **관련 문서:** `1_PRD.md` 섹션 3 (현재 "그랜드마스터/챌린저"로 명시)

## [BACKLOG-006] 리더보드 팀/시작위치별 분리 표시 **(필수)**
* **카테고리:** 프론트엔드 / 기획
* **설명:** 백엔드 DB에는 `[팀(레드/블루) + 시작위치 + 챔피언 + 패치버전]` 복합 키로 기록이 저장된다. 프론트엔드 리더보드에서 이 데이터를 어떻게 보여줄지 결정이 필요하다.
* **선택지:**
  1. 팀/시작위치별로 탭 또는 필터로 분리하여 보여준다.
  2. 통합 랭킹으로 보여주되 팀/시작위치를 라벨로 표기한다.
  3. 가장 빠른 기록만 보여주고 팀/시작위치는 부가 정보로 표시한다.
* **필요 작업:**
  1. UX 방향 결정
  2. PRD 및 API 스펙(`2_API_SPEC.md`)에 `team`, `startPosition` 필드 반영
  3. 프론트엔드 리더보드 UI 수정

## [BACKLOG-007] 신기록 발견 시 실시간 알림
* **카테고리:** 백엔드 / 운영
* **설명:** 배치가 신기록을 발견했을 때 관리자에게 즉시 알림을 보내는 체계. 현재는 배치 주기가 매일이므로 패치 직전 신기록 발견 시 녹화 기회를 놓칠 수 있음.
* **선택지:**
  1. Slack/Discord webhook으로 신기록 알림
  2. 배치 주기를 매일 → 매 N시간으로 단축
  3. 패치 전날에만 배치 주기를 높이는 방식
* **필요 작업:**
  1. 알림 채널 및 방식 결정
  2. 배치에 알림 발송 로직 추가
