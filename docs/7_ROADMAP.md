# 7_ROADMAP.md (JungleClear.gg 로드맵)

## 현재 단계: Phase 1 (진행 중)

---

## Phase 1 — 영상 서빙 웹사이트 구축
유저가 직접 방문하여 챔피언별 정글링 영상을 탐색하고 시청할 수 있는 웹사이트를 완성한다.

**핵심 결과물:**
- 메인 페이지: 챔피언 검색 + 인기 챔피언 그리드
- 상세 페이지: 챔피언 메타 정보 + 영상 리더보드
- Vercel 배포를 통한 실서비스 접근 가능

**현재 상태:**
- ✅ 프론트엔드 구현 완료 (Mock 데이터 기반, 35개 챔피언 + 실제 YouTube 영상)
- ✅ Vercel 배포 준비 완료
- ⬜ 백엔드 API 연동 (현재 Mock → 추후 실데이터 전환)

---

## Phase 2 — 데이터 수집 및 정제
백엔드에서 Riot API를 활용하여 천상계 정글 경기 데이터를 자동으로 수집하고 정제한다.

**핵심 결과물:**
- 매치 데이터 수집 파이프라인 (League API → Match-V5 API → Timeline API)
- 정글 매치 원천 데이터 구축 (`match_record` — SSOT)
- 챔피언별 통계 산출 (승률, 픽률, 밴률)
- 챔피언 간 상성 데이터 구축 (정글 매치업 승률)
- 경기별 룬/소환사 주문 수집
- 순수성 검증 기반 풀캠프 클리어 기록 추출 (`clear_record`) -> 녹화 후보군으로 사용

---

## Phase 3 — 영상 녹화 및 업로드
선별된 경기를 실제 영상으로 녹화하고 YouTube에 업로드한다.

**핵심 결과물:**
- 선별된 경기의 리플레이 영상 녹화 자동화(또는 반자동화)
- YouTube 채널 운영 및 영상 업로드
- 영상 메타데이터 관리 (챔피언, 클리어 타임, 패치 버전, 소환사명 등)

**운영 제약:**
- 롤 클라이언트는 패치 후 이전 리플레이 재생이 불가하므로, 신기록 발견 시 **다음 패치(수요일) 전에 녹화를 완료**해야 한다.
- 초기에는 관리자가 수동으로 리플레이(.rofl) 다운로드 → 녹화 → YouTube 업로드 → `youtubeVideoId`를 DB에 등록하는 방식으로 운영한다.

---

## Phase 4 — 영상 데이터 연동 및 서빙
YouTube에 업로드된 영상 데이터를 웹사이트에 자동으로 연동하여 실서비스를 완성한다.

**핵심 결과물:**
- YouTube Data API를 통한 영상 ID 자동 수집
- DB에 영상 메타데이터 저장 (youtubeVideoId, 클리어 타임, 소환사 정보 등)
- 프론트엔드 Mock 데이터 → 실 API 데이터 전환
- 새 영상 업로드 시 웹사이트 자동 반영
