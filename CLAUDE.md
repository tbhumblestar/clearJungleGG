# JungleClear.gg 프로젝트

## 세션 시작 시 필수 사항
작업을 시작하기 전에 반드시 `docs/0_MASTER_GUIDE.md`를 읽고 프로젝트 맥락을 동기화할 것.

## 프로젝트 구조
- `docs/` — 프로젝트 문서 (PRD, API 스펙, 워크플로우, 배포 설계 등)
- `frontend/` — Next.js 16 프론트엔드 (App Router + Tailwind CSS v4)
- `frontend/DESIGN_SYSTEM.md` — 디자인 시스템 및 컴포넌트 셋
- `design/` — 디자인 에셋 (HTML 프로토타입, 색상 체계)

## 문서 작성 규칙
- **`1_PRD.md`는 유저 경험만 서술한다.** 유저가 무엇을 보고, 어떤 행동을 하고, 어떤 결과를 얻는지. 컴포넌트명, CSS 클래스 등 구현 세부사항은 쓰지 않는다.
- **`3_FRONTEND_IMPLEMENTATION.md`는 화면 구현 방법을 서술한다.** PRD의 유저 경험을 어떤 컴포넌트, 데이터, 상태 관리로 화면을 실현하는지를 단계별로 정의한다.
- 새 기능을 추가할 때는 PRD에 유저 경험을 먼저 정의하고, 프론트엔드 구현서에 구현 방법을 추가한 후, 코드를 작성한다.

## 문서 및 환경 변수 관리 규칙
- **환경 변수 목록은 `docs/0_MASTER_GUIDE.md`에서 관리한다.** `.env`에 변수를 추가/삭제/변경하면 반드시 마스터 가이드의 환경 변수 섹션에도 반영할 것.
- **문서 메타데이터는 `docs/0_MASTER_GUIDE.md`에서 관리한다.** 문서의 제목, 역할, 정체성이 변경되면 반드시 마스터 가이드의 문서 목록을 확인하고 동기화할 것.
